Вариант 8. Табу-поиск для задачи планирования 
----------------------------------------------
 1) Табу-поиск - это это метаэвристический алгоритм оптимизации, предназначенный для решения сложных (часто комбинаторных) задач. Он был предложен Фредом Гловером в 1986 году. Ключевая идея Табу-поиска — это усовершенствованный локальный (или "соседский") поиск. В то время как простой локальный поиск (например, "метод подъема по холму") останавливается, как только достигает локального оптимума (решения, которое лучше всех его "соседей", но не обязательно лучшее вообще), Табу-поиск умеет "выбираться" из таких ловушек.
 2) Принцип работы. Алгоритм работает итеративно, перемещаясь от одного "решения" (S) к другому, "соседнему" (S'), в пространстве всех возможных решений.
    - Начало: Алгоритм стартует с некоторого начального (часто случайного) решения.
    - Окрестность: На каждой итерации он рассматривает "окрестность" текущего решения — набор решений, которые можно получить из текущего с помощью одного "хода" (например, поменяв два города в маршруте или, как в вашей задаче, поменяв рабочий день с выходным).
    - Выбор хода: Алгоритм выбирает лучшее решение из этой окрестности, даже если оно хуже текущего. Это и есть тот механизм, который позволяет "спуститься с холма" локального оптимума, чтобы в итоге найти более высокий "холм" (глобальный оптимум).
    - ​Проблема (Циклы): Разрешение "плохих" ходов создает новую проблему: алгоритм может зациклиться. Например, перейти из решения A в B (потому что B — лучший "сосед"), а на следующем шаге из B вернуться обратно в A (потому что A — лучший "сосед" для B).
