Вариант 8. Табу-поиск для задачи планирования 
----------------------------------------------
 1) Табу-поиск - это это метаэвристический алгоритм оптимизации, предназначенный для решения сложных (часто комбинаторных) задач. Он был предложен Фредом Гловером в 1986 году. Ключевая идея Табу-поиска — это усовершенствованный локальный (или "соседский") поиск. В то время как простой локальный поиск (например, "метод подъема по холму") останавливается, как только достигает локального оптимума (решения, которое лучше всех его "соседей", но не обязательно лучшее вообще), Табу-поиск умеет "выбираться" из таких ловушек.
 2) Принцип работы. Алгоритм работает итеративно, перемещаясь от одного "решения" (S) к другому, "соседнему" (S'), в пространстве всех возможных решений.
    - Начало: Алгоритм стартует с некоторого начального (часто случайного) решения.
    - Окрестность: На каждой итерации он рассматривает "окрестность" текущего решения — набор решений, которые можно получить из текущего с помощью одного "хода" (например, поменяв два города в маршруте или, как в вашей задаче, поменяв рабочий день с выходным).
    - Выбор хода: Алгоритм выбирает лучшее решение из этой окрестности, даже если оно хуже текущего. Это и есть тот механизм, который позволяет "спуститься с холма" локального оптимума, чтобы в итоге найти более высокий "холм" (глобальный оптимум).
    - ​Проблема (Циклы): Разрешение "плохих" ходов создает новую проблему: алгоритм может зациклиться. Например, перейти из решения A в B (потому что B — лучший "сосед"), а на следующем шаге из B вернуться обратно в A (потому что A — лучший "сосед" для B).
 3) Основные функции и компаненты:
    - Начальное решение: Функция, генерирующая стартовую точку (например, случайное расписание).
    - Структура окрестности: Правило, по которому из текущего решения получаются "соседи". (В вашей задаче: поменять один рабочий день и один выходной у одного сотрудника).
    - Функция оценки (Objective Function): "Оценка качества" из вашего задания. Это функция, которая вычисляет, "насколько" хорошо или плохо текущее решение. В задачах с ограничениями это часто штрафная функция. Цель — минимизировать штраф.
    - Табу-лист: Память для хранения запрещенных ходов.
    - Аспирационный критерий: Правило, отменяющее табу (например, если новое решение — рекордно хорошее).
    - Критерий остановки: Условие завершения поиска (например, достигнуто 0 штрафа, или прошло 1000 итераций).
 4) Временная сложность - O(k * |N| * C), где:
    - k — количество итераций (задается критерием остановки).
    - |N| — размер окрестности (количество "соседей", которых мы проверяем на каждой итерации).
    - C — стоимость вычисления функции оценки (штрафа) для одного "соседа"
 6) Объяснение временной сложности. На каждой из k итераций алгоритм в своем базовом варианте должен:
      1) Сгенерировать (или, по крайней мере, оценить) всех соседей в окрестности |N|.
      2) Для каждого соседа рассчитать его "оценку качества", что стоит C.
      3) ​Проверить, не является ли ход к этому соседу "табу" (обычно это быстрая операция, O(1) или O(длина_табу_листа)).
      4) Выбрать лучшего из всех не-табуированных (или аспирационных) соседей.
    Самая "тяжелая" часть внутри цикла — это оценка всей окрестности, которая стоит O(|N| * C). Повторяя это k раз, мы и получаем итоговую сложность.
